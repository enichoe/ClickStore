⚡ Fases de Implementación

FASE 1: PREPARACIÓN (30 minutos)

  1. Hacer backup de tu base de datos Supabase
  2. Iniciar en modo "staging" (no producción aún)
  3. Leer archivo: ARQUITECTURA-SEGURIDAD-PRODUCCION.md

FASE 2: EJECUTAR SQL (1-2 horas)

2.1 Ejecutar: supabase-rls-production.sql

  1. Ve a: Supabase Dashboard → SQL Editor
  2. Crear Nueva Query
  3. Copiar contenido completo de supabase-rls-production.sql
  4. Ejecutar (☐ Verificar que no haya errores)
⚠️ IMPORTANTE:
  • Esto REEMPLAZARÁ las políticas RLS antiguas
  • Si algo falla: SELECT * FROM information_schema.role_table_grants;
  • Garantiza backups automáticos habilitados

2.2 Ejecutar: supabase-orders-functions.sql

  1. Nueva Query en SQL Editor
  2. Copiar contenido de supabase-orders-functions.sql
  3. Ejecutar (☐ Verificar funciones en Functions → Database Functions)
✅ Verificación:
En Supabase Dashboard, deberías ver:
  • Funciones: create_order_secure, update_order_status, etc.
  • Tablas: store_members, audit_logs, alerts
  • Índices nuevos sobre stores, products, orders

FASE 3: INTEGRAR JAVASCRIPT (2-3 horas)

3.1 Agregar archivos JS al index.html

En tu index.html, dentro de <head> o antes de </body>:

<!-- 1. Image Optimizer (debe cargar primero) --> <script src="js/image-optimizer.js"></script> <!-- 2. Security Validators --> <script src="js/security-validator.js"></script> <!-- 3. Order Handler Seguro (integración con lógica existente) --> <script src="js/order-handler-secure.js"></script>

3.2 Reemplazar funciones existentes en js/auth.js

Donde actualmente tienes función handleRegister(), después de crear la tienda:

// Crear tienda const { data: storeData, error: storeError } = await supabase .from('stores') .insert([...]) .select() .single(); // NEW: Inicializar miembro de tienda como 'owner' if (storeData && !storeError) { const { error: memberError } = await supabase .from('store_members') .insert([{ store_id: storeData.id, user_id: user.id, role: 'owner', permissions: { 'view_orders': true, 'edit_products': true, 'edit_store': true, 'update_orders': true } }]); if (memberError) { console.error('Error creating store member:', memberError); } }

3.3 Reemplazar función de crear orden

En tu js/storefront.js (o donde manejas checkout):

// ANTES (inseguro): async function submitOrder() { const result = await supabase .from('orders') .insert([...]); // ❌ Sin validación } // DESPUÉS (seguro): async function submitOrder() { const result = await createOrderSecure( appState.tenant.id, appState.cart, document.getElementById('customer-name').value, document.getElementById('whatsapp').value ); }

3.4 Reemplazar upload de imágenes

En js/admin.js (donde subes productos):

// ANTES (sin optimización): async function handleImageUpload(file) { const { data, error } = await supabase.storage .from('product-images') .upload(`path/${file.name}`, file); } // DESPUÉS (con optimización): async function handleImageUpload(file) { const result = await uploadProductImageSecure( file, appState.tenant.id, productId ); if (result.success) { // Guardar result.paths en BD } }

🆘 Troubleshooting

Problema: "Permission denied" en RLS

Causa: Políticas antiguas conflictando
Solución:

-- Ejecuta en SQL Editor: DROP POLICY IF EXISTS "public_select_stores_by_slug" ON public.stores; DROP POLICY IF EXISTS "owner_insert_stores" ON public.stores; -- ... (eliminate all old policies) -- Luego re-ejecuta supabase-rls-production.sql

Problema: "Function not found" en RPC

Causa: supabase-orders-functions.sql no ejecutado
Solución: Ejecutar SQL en Supabase Dashboard → SQL Editor

Problema: Imágenes no se cargan después de upload

Causa: Rutas de Storage incorrectas
Solución:

// Verificar rutas generadas: console.log('Paths:', result.paths); // Debe ser: "stores/[uuid]/products/full/[hash].webp" // Verificar URL completa: const url = imageHelper.getImageUrl(path); console.log('Full URL:', url);
